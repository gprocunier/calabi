# Calabi v1.0.0 (2026-04-05)

This is the first tagged release of Calabi.

Calabi is an Ansible-driven, single-host, fully disconnected OpenShift 4 lab
built on nested KVM. It is designed to let you demonstrate and iterate on
production-patterned installer and day-2 workflows while keeping the
infrastructure shape realistic.

## What’s Included

- `aws-metal-openshift-demo/`: the main lab implementation (AWS scaffolding,
  hypervisor bootstrap, support guests, disconnected OpenShift install, day-2).
- `cockpit/calabi-observer/`: Cockpit plugin providing real-time observability
  for the host resource management system on `virt-01` (RPM spec included).
- Documentation map and deep-dive guides under `aws-metal-openshift-demo/docs/`.

## Validated Baseline (As Documented)

- OpenShift `4.20.15`
- 3 masters: `8 vCPU / 24 GiB`
- 3 infra: `16 vCPU / 48 GiB`
- 3 workers: `4 vCPU / 16 GiB` (with guidance for uplift to `8 vCPU / 16 GiB`)

See `aws-metal-openshift-demo/docs/prerequisites.md` for current controller and
input expectations (including `ansible-core 2.18` and a RHEL `10.1` guest image
source).
